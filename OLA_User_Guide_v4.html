1. 前言

1.1 OLA 是什么?

OLA 是一个帮您把跨境 B2B 销售订单"从询盘到成交、再到续约"全流程跑完的 AI 助手。

  1. 听:听懂您和客户在电话里、微信里、邮件里说了什么(支持普通话 / 粤语 / 英语,以及它们的混杂)
  2. 记:把每次互动自动整理进每个客户的"持续画像"——客户是谁、要什么、在乎什么、决策风格如何
  3. 干:替您写报价单、回询盘、做跟进、提醒续约——并且报价的每一分钱都来自您真实的产品目录,不是 AI 编的
  4. 教:在您打电话、写邮件的时候,实时提示您"客户刚才提的这个顾虑,建议这样回应"

1.2 OLA 与其他销售工具有什么不一样?

Ola和 Salesforce、HubSpot、SAP、纷享销客、销售易, Gong、Fireflies有什么不一样?

问题OLA
手动录入，数据残缺自动从通话/邮件/WhatsApp 抓取
客户资料是通话索引，新人难接手持续客户画像，30 秒上手
只给洞察，报价还得自己写直接从真实目录生成报价单，一键发送
只支持英文普通话 / 粤语 / 英语混杂原生支持
多工具拼凑，成本叠加一个 agent 跑通询盘到续约，成本约十分之一
AI 报价会编数字，销售不敢用确定性报价，每个数字可追溯，零幻觉

1.3 OLA 的三层架构

OLA 是由三层能力组成的完整系统

│  ┌────────┬────────┬──────────┬──────────┬──────┐  │

└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐

                         ↓
┌─────────────────────────────────────────────────────┐
│  L3 · 执行自动化                                     │
│  根据客户在销售漏斗的位置,OLA 替您干不同的活儿       │
│  ┌──────────┬──────────┬──────────┬──────────┐     │
│  │ 询盘应答 │ 报价生成 │ 实时教练 │ 续约外联 │     │

  • L1 · 数据Data:自动抓取通话、邮件、WhatsApp 里的信息,销售不用改变任何工作习惯。

2. 智能助手 AskOLA

AskOLA 是您日常和 OLA 打交道的主要入口。无论您想查信息、改资料、新建记录,都可以在这里用一句话搞定,不用点菜单、不用学操作。

接下来三章,我们用最朴素的方式教您:

  • 第一章:用 AskOLA 查询信息(查客户、查商品、查报价单,查画像、查通话)
  • 第二章:用 AskOLA 修改信息(改资料、改价格、改报价单)
  • 第三章:用 AskOLA 录入新信息(建客户、建商品、建报价单)

2.1 查询信息

这一章教您如何用 AskOLA 快速查到您需要的客户资料、商品信息、报价单,以及更深层的客户画像和通话记录。

2.1.0 打开 AskOLA

登录 OLA 系统后,在页面右下角会看到一个对话框图标,点击它,AskOLA 聊天窗口就打开了。

您可以在系统任意页面随时打开它,不影响您正在做的其他操作。

🟡 路线图 (v4): AskOLA 将支持桌面快捷键(Ctrl+K / Cmd+K)、浏览器扩展、以及移动端独立 App,让您在任何场景下都能 30 秒内问到答案。

2.2 修改信息

这一章教您如何通过 AskOLA 直接修改系统里已有的记录,不用进入表单页面一个字段一个字段地找。

⚠️ 重要提醒: 每次修改前,系统都会把改动内容展示给您确认一遍,您点"确认"之后修改才会生效。所有修改都有操作记录,可以事后查看是谁在什么时候改的。

2.2.1 修改客户信息

什么时候用这个功能?

  • 客户换了联系人或联系方式
  • 想更新客户的跟进状态(比如从"在谈中"改为"已成交")
  • 想给客户加一条备注(比如记录特殊要求)
  • 发现之前录入的信息有错误,需要更正

怎么操作?(以修改联系方式为例)

第一步: 在 AskOLA 对话框里,用自然语言告诉系统您想改什么。

把 XX 公司的联系电话改成 138XXXXXXXX

第二步: 系统找到这条客户记录,显示当前信息,并问您是否确认修改:

已找到客户"XX 公司",当前联系电话为:136XXXXXXXX
您要修改为:138XXXXXXXX
确认修改吗?【是】 【否】

第三步: 点击"是",修改立即生效。系统同时记录:操作人、修改时间、改了哪个字段、改成了什么。

更多修改示例

改联系人邮箱:

把 XX 公司的邮箱改成 abc@example.com

改跟进状态:

把 XX 公司的状态改成"已成交"

加备注:

给 XX 公司加一条备注:客户要求每次发货前提前三天通知

改客户所在地区:

把 XX 公司的地区改成"华东"

注意事项

  • 如果您没有修改权限,系统会提示"您没有编辑权限,请联系管理员"
  • 已经删除的客户记录无法恢复,删除前请仔细确认

2.2.2 修改商品信息

什么时候用这个功能?

  • 原材料涨价,需要更新售价
  • 进了一批货,需要更新库存数量
  • 产品规格有调整,需要修改描述
  • 某款产品停售,需要把它下架

怎么操作?(以修改价格为例)

第一步: 告诉系统您要改哪款产品的什么内容。

把无线温度传感器的单价从 68 元改成 75 元

第二步: 系统显示当前信息并请您确认:

已找到"无线温度传感器",当前单价:68 元
您要修改为:75 元
⚠️ 提示:此修改将影响所有未发送的新报价单;已发送的报价单不受影响。
确认修改吗?【是】 【否】

第三步: 点击"是",修改生效。

更多修改示例

改库存数量:

无线温度传感器刚到货了,库存更新为 500 件

修改产品描述:

给无线温度传感器的说明里加一句:适用工作温度 -40°C 至 125°C

下架产品:

把"旧款压力传感器"设为停用,这款已经不卖了

修改阶梯价格:

把无线温度传感器改成阶梯定价:不足 100 件单价 75 元,100 件及以上单价 68 元

💡 提示: 如果阶梯定价条件比较复杂,直接输入您的定价规则,系统会引导您一步步确认每个档位。

注意事项

  • 修改价格不会自动更新您已经发出去的报价单,如果报价单也需要改,请参考 2.3 节
  • 产品下架后,它不会出现在新的报价单里,但历史记录里仍然保留,不影响过去的数据

2.2.3 修改报价单

什么时候用这个功能?

  • 客户要求改数量、换产品
  • 需要给客户加折扣
  • 交货期或付款条件有变化
  • 报价单快过期了,需要延长有效期
  • 发现报价单里有录入错误

怎么操作?(以修改数量为例)

第一步: 指定要修改哪份报价单,以及要改什么。

把给 XX 公司最新那份报价单里,无线温度传感器的数量从 200 件改成 350 件

第二步: 系统调出报价单,显示修改前和修改后的对比,请您确认:

报价单编号:2025-0312,客户:XX 公司
无线温度传感器:200 件 → 350 件
单价根据阶梯定价规则,从 68 元自动调整为 65 元(达到 300 件及以上档位)
金额变化:13,600 元 → 22,750 元
确认修改吗?【是】 【否】

第三步: 点击"是",系统保存新版本,同时保留修改前的历史版本。

更多修改示例

加折扣:

给 XX 公司最新的报价单打九折

改付款方式:

把 2025-0298 这份报价单的付款条件改成:30% 预付,货到后付剩余 70%

延长有效期:

把 2025-0312 这份报价单的有效期延长到 6 月 30 日

新增一个产品:

在给 XX 公司的报价单里加上"工业压力传感器"50 件

删除一个产品:

把 2025-0312 这份报价单里的"旧款压力传感器"去掉

注意事项

  • 每次修改都会保存一个新版本,您可以在报价单详情页里点"历史版本"查看修改记录
  • 如果报价单已经被客户确认签收,部分内容会被锁定,需要联系管理员才能解锁修改
  • 修改完成后,如果需要重新发给客户,可以直接在 AskOLA 里输入"把这份报价单重新发给 XX"

2.3 录入新信息

这一章教您如何通过 AskOLA 新建客户档案、新增商品、新建报价单,不用手动填写表单,直接跟系统说就行。

2.3.1 新建客户档案

什么时候用这个功能?

  • 参加展会拿到名片,回来要批量录入客户
  • 收到新客户发来的询盘,需要先把对方信息存进系统
  • 以前用别的方式管理客户,现在要把老客户迁移进来

必须提供的信息

  • 客户公司名称(必填)
  • 联系人姓名(必填)
  • 联系方式:手机号或邮箱,至少填一个(必填)

可以补充的信息(不填也没关系,以后可以再加)

  • 客户所在国家 / 省市地区
  • 客户所在行业
  • 这个客户是从哪里来的(展会 / 朋友介绍 / 网络推广等)
  • 备注(任何您觉得有用的信息)

怎么操作?

方式一:一次性说清楚

帮我录入一个新客户:XX 贸易有限公司,联系人李总,手机 138XXXXXXXX,做电子元器件批发,在上海

系统收到后,会显示录入预览,您确认无误后点"保存",客户档案就建好了。

方式二:让系统引导您填

帮我新建一个客户档案

系统会一步步问您:客户名称是什么?联系人叫什么?怎么联系?……您一步步回答就行。

方式三:从邮件或消息里自动提取(推荐)

如果客户发来了一封自我介绍的邮件,或者在微信 / 邮件里留了联系方式,您可以把那段文字直接复制粘贴到对话框,然后输入:

从这段信息里帮我提取客户资料并建档

系统会自动识别里面的公司名、联系人、电话、邮箱等信息,生成一份草稿让您确认。

🟡 方式四 (v4):从名片照片提取

直接拍一张名片照片,上传后输入:

这张名片帮我建档

系统会自动识别名片上的所有信息,生成草稿。

🔵 方式五(长期愿景):从展会邮件批量建档

展会结束后,把您所有收到的询盘邮件转发到一个专属地址,OLA 自动逐封识别、批量建档、自动归类,并把这些新客户加入相应的跟进序列。

注意事项

  • 如果系统发现您要录入的客户名称和已有客户很相似,会提示您"是否是同一个客户",避免重复建档
  • 确认保存后,客户档案立即生效,可以在客户列表里找到
  • 新建的客户档案会自动进入 L2 画像系统,从下一次互动开始,OLA 就会自动维护它的画像

2.3.2 新增商品

什么时候用这个功能?

  • 有新产品要上架
  • 从供应商那里拿到了新产品报价,需要录进系统
  • 扩充产品目录

必须提供的信息

  • 产品名称(必填)
  • 产品货号(必填,如果您公司有自己的编号规则)
  • 销售价格(必填,至少填一个基础单价)

可以补充的信息

  • 产品说明 / 规格描述
  • 产品所属类别(方便以后按类别查找)
  • 当前库存数量
  • 阶梯定价规则(例如:不足 100 件单价 75 元,100 件以上单价 68 元)
  • 产品图片地址(如有)
  • 🟡 利润底线(销售报价时的最低许可价)
  • 🟡 适用场景标签(便于客户问"我要做户外监测"时系统自动推荐)

怎么操作?

一次性录入:

新增一款产品:无线温度传感器,货号 WTS-2024,单价 68 元,库存 500 件,属于工业传感器类别

让系统引导:

帮我录入一款新产品

系统会逐项问您需要填的信息。

批量录入多款产品:

把产品信息整理成一个列表(比如从表格里复制出来),粘贴到对话框,告诉系统:

帮我把下面这些产品都录入系统:
[粘贴产品列表]

系统会逐条识别并请您确认,每条确认后再进行下一条,避免出错。

🟡 从供应商报价单 PDF 导入 (v4):

直接把供应商发来的 PDF 报价单拖入 AskOLA:

帮我把这份供应商报价单里的产品都加进我的产品目录

系统自动识别 PDF 里的产品信息,生成草稿目录,您只需要补充自己的售价和分类。

注意事项

  • 录入完成后,这款产品就会出现在产品目录里,做报价单的时候可以直接调用
  • 如果价格录入有误,后续可以通过"修改商品信息"功能修正,不影响已经发出的报价单

2.3.3 新建报价单

什么时候用这个功能?

  • 客户发来询盘,要快速给对方出一份报价
  • 要为重要客户做一份特殊报价
  • 想复制上次给某个客户的报价,改一改数量再发一次

必须提供的信息

  • 哪个客户(系统里已有的客户名称)
  • 要报价哪些产品、各需要多少数量

可以补充的信息

  • 交货期(例如:30 天)
  • 付款方式(例如:电汇,30% 预付)
  • 报价有效期(例如:有效期 15 天)
  • 特殊折扣或优惠说明
  • 备注

怎么操作?

一次性说清楚(推荐):

给 XX 公司做一份报价:无线温度传感器 200 件、工业压力传感器 100 件,付款方式电汇,有效期 30 天

系统收到后,自动从产品目录里取出价格,生成一份报价单草稿,并显示给您预览:

已为 XX 公司生成报价单草稿:
· 无线温度传感器 × 200 件,单价 68 元,小计 13,600 元
· 工业压力传感器 × 100 件,单价 120 元,小计 12,000 元
· 合计:25,600 元
· 付款方式:电汇
· 有效期:30 天
确认保存吗?【保存草稿】 【保存并发送】

让系统引导:

帮我给 XX 公司做一份报价

系统会一步步问您要报哪些产品、数量多少。

基于历史报价复制:

复制上次给 XX 公司的报价单,产品不变,数量全部改成两倍

系统调出上次的报价单,按您的要求调整后展示给您确认。

关于价格的重要说明:零幻觉报价

AskOLA 生成报价单时,所有价格都来自您自己录入的产品目录,系统不会自己估算或猜测价格。这是 OLA 的核心承诺——确定性报价(Deterministic Quoting)。

具体来说:

  • 每一个 SKU 都是真实的:OLA 不会"想象"出一个您目录里没有的产品
  • 每一个单价都有出处:可以追溯到您产品目录里的具体记录
  • 每一笔小计都是数学计算,不是 AI 估算
  • 阶梯定价、折扣、税费的计算全部基于您预设的规则,可重现、可审计

如果某款产品还没有设定售价,系统会直接提示:

⚠️ "无线温度传感器(WTS-2024-Pro)"在您的产品目录里没有售价,请先去产品目录补充售价,再生成报价单。

为什么这件事重要?

很多 AI 销售工具会"编"价格——表面上能快速生成报价,但销售不敢直接发给客户,因为不知道哪个数字是 AI 想象的。OLA 的设计哲学是:AI 帮您写表达,数据全部来自真实目录。您可以放心地一键发送。

报价单保存后怎么发给客户?

报价单默认保存为草稿,不会自动发送。您可以:

  • 保存后手动发送(推荐,发送前可以再检查一遍)
  • 或者在保存时直接选择"保存并发送"
  • 🟡 (v4) 设置"自动发送规则"——比如"对于历史成交超过 10 次的老客户,生成草稿后 5 分钟内自动发送,除非我手动取消"

3.1 通话录音上传与分析

这一章教您如何把与客户的通话录音上传到 OLA 系统,让 OLA 帮您自动整理通话内容、找出值得改进的地方,帮助您和团队不断提升谈单能力。

3.1.1 功能说明

做销售,很多时候跟客户打完电话,转头就忘了说了什么。或者明明觉得聊得不错,但客户最后没下单,也说不清楚哪里出了问题。

OLA 的录音分析功能就是为了解决这个问题。

您只需要把通话录音上传进来,系统会帮您:

  • 自动把录音转成文字,完整保留通话内容,再也不用手动记录
  • 自动识别通话里的关键信息,比如客户提到的产品需求、顾虑、预算、交货要求等
  • 自动分析哪些地方做得好、哪些地方可以改进,比如客户提出价格顾虑时您的回应是否有效
  • 结合您公司的销售经验和成功案例,给出具体的改进建议,就像有一位有经验的老业务在旁边帮您复盘
  • 🟡 自动更新客户画像:这次通话里出现的新信息(新顾虑、新需求、新决策人)会自动合并进客户画像

举个例子:
您打完一个客户电话,上传录音后,系统告诉您:"客户在第 3 分钟提到对交货期有疑虑,但您当时直接跳过去继续介绍产品了,建议下次先正面回应交货期问题,再继续推进。"——这就是系统帮您做的事。

3.1.2 上传前准备

上传录音前,请确认以下几点:

  1. 录音文件格式:支持常见格式,包括 .mp3、.m4a、.wav、.aac(微信语音导出的文件也支持)
  2. 录音文件大小:单个文件不超过 500MB(普通通话录音远低于这个限制,不用担心)
  3. 录音时长:建议上传完整的通话录音,最短不低于 1 分钟,太短的内容系统无法有效分析。对于超长录音(超过 25 分钟),系统会自动分块处理,您不用做额外操作。
  4. 通话语言:目前支持普通话、英语、粤语,以及它们的混杂(对跨境贸易很常见)。其他方言暂不支持(如通话中混有大量方言,转写准确率可能下降)

💡 提示: 大多数手机自带录音功能,通话时可以同步录音。如果您的手机不支持,可以咨询管理员推荐合适的录音工具。

3.1.3 上传方式

方式一:通过 AskOLA 上传

第一步: 打开 AskOLA 对话框,输入:

我要上传一段通话录音

第二步: 系统弹出文件上传窗口,点击"选择文件",找到您手机或电脑里的录音文件,点击上传。

第三步: 系统询问这段录音是跟哪个客户的通话:

请问这段录音是与哪位客户的通话?(请输入客户名称)

输入客户名称后,系统自动将这段录音与该客户档案关联,方便以后查找。

第四步: 上传完成后,系统显示:

录音已上传成功,正在转写和分析,预计需要 3 到 5 分钟,完成后会通知您。

方式二:在客户档案页直接上传

打开某个客户的档案页,找到"通话记录"标签,点击"上传录音"按钮,选择文件上传即可。上传后录音自动归属于该客户,无需再次确认。

3.1.4 分析报告内容

录音处理完成后,系统会生成一份通话分析报告,包含以下几个部分:

一、通话内容摘要

系统把整段录音转成文字后,提炼出一段 200 字左右的摘要,告诉您这次通话主要聊了什么,方便您快速回顾。

示例:

本次通话时长 12 分 34 秒。客户主要询问了无线温度传感器的交货周期和最低起订量,对价格表示接受,但希望能将交货期从 30 天缩短至 20 天。通话结束时客户表示需要内部确认后再回复。

二、客户需求与关键信息提取

系统自动从通话中识别并整理出客户提到的重要信息:

类别客户说的内容
需要的产品无线温度传感器,规格 XX
数量需求首批 300 件,后续视情况追加
价格接受情况对现有报价无异议
主要顾虑交货期能否缩短到 20 天以内
下一步行动等客户内部确认,预计三天内回复

这些信息可以一键同步到该客户的档案备注里,不用手动再记一遍。

三、通话亮点

系统标注出这次通话里做得好的地方,帮您知道哪些方式是有效的,值得继续保持。

示例:

✅ 您在介绍产品时主动提到了竞品的对比优势,这对客户的判断很有帮助。
✅ 您在客户提出顾虑后,没有急于反驳,而是先表示理解再给出解释,节奏把握得当。

四、改进建议

系统结合公司积累的销售经验,指出这次通话中可以做得更好的地方,并给出具体的改进方向。

示例:

⚠️ 客户在第 4 分 12 秒提到"交货期能不能再快一点",您当时的回应是"我去问一下",但没有给出一个明确的时间节点。建议下次碰到类似情况,先给一个初步答复(比如"最快可以做到 25 天,我去再确认一下能否缩短到 20 天"),让客户感觉到您在积极推进,而不是不确定。

⚠️ 通话结束前,您没有明确约定下一次跟进的时间。建议结束通话前主动说:"那我后天下午联系您确认一下内部情况,方便吗?"——这样可以有效减少客户拖延回复的情况。

3.1.5 查看历史录音

方式一:通过 AskOLA 查询

帮我查一下上周和 XX 公司的通话分析报告

系统显示该次通话的完整分析报告,您可以选择查看全文或只看改进建议。

方式二:在客户档案里查看

打开某个客户的档案,点击"通话记录"标签,可以看到与该客户的所有历史录音,点击任意一条可以查看详细分析报告。

方式三:查看团队整体情况(仅限管理员)

管理员可以在"团队数据"页面,查看所有销售人员的通话记录和分析汇总,了解团队整体的通话质量和常见的改进点。详见第十二章。

3.1.6 注意事项

  • 录音内容仅供内部使用,系统不会将您的通话内容分享给任何第三方
  • 建议在征得客户同意后再录音,部分地区对录音有相关法律规定,请遵守当地规定
  • 如果录音质量较差(背景噪音大、信号断断续续),转写准确率可能下降,分析结果仅供参考
  • 上传后的录音文件可以在客户档案里删除,删除后对应的分析报告也会一并移除,且无法恢复

5. 故障与求助

5.1 常见问题与故障排查

5.1.1 使用类常见问题

我输入的问题,AskOLA 没看懂,怎么办?

换一种说法再试一次。比如查不到,可以换用客户的全称,或者换成货号来查。如果还是不行,可以联系技术支持(见文末)。

💡 一个小技巧:如果一个问题问了 3 次都没得到想要的结果,直接换更简短、更具体的关键词,反而比写更长的句子有效。

我改错了,能不能撤销?

修改操作生效后无法一键撤销。但是每次修改系统都有记录,您可以联系管理员查看修改历史,并手动把内容改回来。所以每次修改前,请仔细看系统显示的"确认修改"提示。

AskOLA 会不会把我公司的数据泄露出去?

不会。系统里所有的客户、商品、报价信息都只存在您公司自己的数据库里。AskOLA 在处理您的数据时,所有传输都经过加密;您的客户对话也不会被用于训练 AI 模型(详见第十四章)。

我没有权限做某个操作,怎么办?

联系您公司的系统管理员,让他给您的账号开通相应的权限。

报价单能直接发给客户吗?

可以。报价单确认无误后,您可以通过系统直接发送给客户(发到对方邮箱、WhatsApp 等),也可以下载成文件自己发送。

录音上传后多久能看到分析报告?

一般 3-5 分钟。如果录音超过 30 分钟,可能需要 10 分钟以内。如果超过 15 分钟还没出报告,请联系技术支持。

我可以同时上传多个录音吗?

可以。一次最多 10 个文件,系统会逐个处理,每个文件的分析报告独立生成。

AskOLA 能听懂方言吗?

目前支持普通话、英语、粤语,以及它们的混杂。其他方言(闽南语、上海话等)暂不支持。

5.1.2 报错与异常排查

录音上传失败

可能原因:

  1. 文件格式不支持 — 请确认是 .mp3 / .m4a / .wav / .aac 之一
  2. 文件超过 500MB — 请压缩后再上传,或联系管理员
  3. 网络不稳定 — 换个网络环境再试

报价单生成时提示"产品没有价格"

说明您的产品目录里这款产品还没有设定售价。请先去产品目录补充该产品的价格,再生成报价单。这是 OLA 的"零幻觉"原则——系统不会自己估算价格。

客户画像里某些信息看起来不对

画像是基于互动数据自动生成的,可能存在误判。您可以:

  1. 直接告诉 AskOLA:"XX 公司的价格敏感度应该是低,不是中等"
  2. 系统会修正,并在以后类似场景下学习您的判断

系统响应很慢

可能原因:

  1. 您的网络较慢
  2. 该时段使用人数较多
  3. 系统在处理较复杂的查询(比如跨多个客户的对比)

如果持续超过 30 秒无响应,请联系技术支持。

A.1 录音相关问题

A1.1 用户反馈"录音上传失败 / 卡住"

给用户的建议(可直接回复):

  1. 先检查文件格式是不是 .mp3 / .m4a / .wav / .aac
  2. 检查文件大小是否超过 500MB
  3. 换个网络环境再试一次
  4. 如果还是不行,请联系技术支持,并告诉我们:文件大小、格式、报错的具体信息

可能的原因(内部):

  • 文件超过 OpenAI Whisper 单次请求的 25MB 限制(需 chunking)
  • 文件超过 1400 秒时长上限(需自动分块,见 chunking 模块)
  • 上传过程中网络中断
  • 文件编码异常(罕见,通常是用户用了非标准录音工具)

内部 debug 路径:

  1. 查 /api/upload 接口日志,看返回的具体错误码
  2. 如果是 413 错误 → chunking 模块未正确处理大文件,检查 chunking pipeline
  3. 如果是 415 错误 → 文件格式识别失败,检查 mime-type 识别
  4. 如果是 500 错误 → 看 backend 异常栈
  5. 如果上传成功但分析未启动 → 检查后台任务队列状态

相关代码位置:

  • 上传:backend/api/upload_audio.py
  • 分块:backend/services/chunking.py(注意 1400s 上限)
  • 转写:backend/services/transcription.py(对接 Whisper API)

A1.2 用户反馈"录音上传成功但看不到分析报告"

给用户的建议:

  1. 请等待 5-10 分钟(超长录音可能需要更久)
  2. 刷新页面看一下
  3. 如果超过 15 分钟还没出报告,告诉我们录音的客户名称和上传时间

可能的原因:

  • 后台分析任务卡住
  • Whisper API 报错或超时
  • 分析 LLM 请求失败(注意当前用的是 Gemini 3.1 Flash)
  • 任务队列积压

内部 debug 路径:

  1. 在任务队列后台查找该用户的任务 ID
  2. 看任务状态:queued / processing / failed
  3. 如果是 failed,看错误信息
  4. 如果是 processing 但超过 10 分钟 → 可能卡在某一步,看具体日志
  5. 如果队列积压 → 检查 worker 数量

相关代码位置:

  • 任务调度:backend/services/task_queue.py
  • 分析 pipeline:backend/services/analysis_pipeline.py
  • LLM 接入:backend/integrations/gemini_client.py

A1.3 用户反馈"录音转写不准 / 听不清楚"

给用户的建议:

  1. 录音背景噪音是不是比较大?
  2. 通话双方是不是有信号不稳的情况?
  3. 是不是有大量方言混入?
  4. 我们可以试着用另一种转写策略重新分析,需要您把录音重新提交一次

可能的原因:

  • 录音质量本身差
  • 包含 OLA 当前不支持的方言
  • 多人同时说话造成 ASR 困难
  • Whisper 模型在特定口音上的表现不佳

内部 debug 路径:

  1. 听一下原始录音,主观判断质量
  2. 看转写结果,识别问题片段
  3. 考虑是否需要换用其他 ASR 服务(例如阿里云 / 腾讯云的中文 ASR 对粤语更友好)

A.2 提问与查询相关

A2.1 用户反馈"AskOLA 听不懂我的问题"

给用户的建议:

  1. 换用更短、更具体的关键词
  2. 加上具体的客户名、产品名、日期
  3. 试试"列出..."、"查询..."、"修改..." 开头的句式

可能的原因:

  • 用户使用了模糊指代("那个客户"、"上次那个")
  • 关键词在系统里没有对应实体
  • LLM 上下文窗口过载(对话太长)
  • Skill 加载失败,LLM 没有相关知识

内部 debug 路径:

  1. 看用户的原始 query
  2. 看 LLM 接到 query 时加载了哪些 skill / context
  3. 如果 skill 未加载 → 检查 skill 触发规则
  4. 如果 skill 加载但回答不对 → 看 LLM 的 reasoning 过程

A2.2 用户反馈"查询结果不全 / 漏了客户"

给用户的建议:

  1. 试试用客户全称查
  2. 试试换个字段(用电话、邮箱查)
  3. 是不是您只有部分客户的查看权限?

可能的原因:

  • 客户实体匹配失败(用户输入的简称未关联到全称)
  • 用户权限不足,系统已过滤
  • 数据库索引未覆盖某些字段
  • 客户合并/拆分历史造成查询断裂

内部 debug 路径:

  1. 直接在数据库查该客户是否存在
  2. 看用户的权限配置
  3. 看 query 转 SQL 时是否覆盖了所有相关表

A.3 报价单相关

A3.1 用户反馈"报价单价格不对"

给用户的建议:

  1. 价格是从您产品目录里取的,先检查目录里这款产品的价格
  2. 如果有阶梯定价,确认数量对应的档位
  3. 如果有客户折扣,确认折扣规则是否正确设定

可能的原因:

  • 产品目录价格被修改但报价单未刷新
  • 阶梯定价规则配置错误
  • 客户折扣应用错误
  • 汇率换算问题(跨境场景)

内部 debug 路径:

  1. 取该报价单的"可追溯审计页"(见第九章 9.4)
  2. 逐项对照:每个价格的数据源是什么
  3. 如果某项与目录不一致 → 看是否目录最近被修改
  4. 如果是规则应用错误 → 看规则引擎日志

相关代码位置:

  • 报价单生成:backend/services/quote_generation.py
  • 定价规则引擎:backend/services/pricing_engine.py
  • 必须 100% 来自目录,不允许 LLM 估算

A3.2 用户反馈"报价单生成失败"

给用户的建议:

  1. 您指定的所有产品是不是都已经在目录里、且都设置了价格?
  2. 客户档案是不是完整(至少有公司名和联系方式)?
  3. 把您的报价请求原话发给我们,我们看看是哪一步出了问题

可能的原因:

  • 产品 SKU 在目录中不存在
  • 产品没有价格设置
  • 客户不存在或信息不全
  • LLM 解析需求时出错(意图识别失败)

内部 debug 路径:

  1. 看用户原始 query
  2. 看 LLM 意图解析的输出
  3. 看产品匹配的结果
  4. 看价格查询的结果

A.4 画像与客户相关

A4.1 用户反馈"画像信息不对"

给用户的建议:

  1. 您可以直接告诉 AskOLA 正确的信息,例如"XX 公司价格敏感度应该是低"
  2. 系统会修正,并在以后类似场景下学习

可能的原因:

  • LLM 在某次互动中误判
  • 用户输入的互动记录本身有歧义
  • 画像更新规则需要调优

内部 debug 路径:

  1. 找到该画像的源数据(哪些互动支持了这条判断)
  2. 评估 LLM 在该场景下的判断是否合理
  3. 如果是系统性问题 → 考虑调整 prompt 或加入新的判断规则

A4.2 用户反馈"两个客户其实是同一个,系统没识别"

给用户的建议:

  1. 您可以在客户列表里手动合并两条记录
  2. 操作:进入任一客户档案 → "合并"按钮 → 选择另一个客户 → 确认

可能的原因:

  • 自动匹配的强匹配规则未触发(没有完全相同的电话/邮箱/公司名)
  • 弱匹配规则置信度不足以触发提示
  • 数据本身差异过大(不同语言、不同主体名)

内部 debug 路径:

  1. 查两个客户的所有字段,看哪些可以作为匹配线索
  2. 调整匹配阈值或增加新的匹配维度

A.5 系统性问题

A5.1 用户反馈"系统很慢"

给用户的建议:

  1. 试试刷新页面
  2. 检查您的网络
  3. 如果持续慢,告诉我们大致时间,我们查一下

可能的原因:

  • 用户端网络问题
  • 服务器负载
  • 数据库慢查询
  • LLM API 响应慢(Gemini Flash 有时会变慢)

内部 debug 路径:

  1. 看系统监控:CPU / Memory / DB 连接数
  2. 看 LLM API 调用日志:是否有超时
  3. 看具体接口的 P95 / P99 响应时间

A5.2 用户反馈"页面打不开 / 报错"

给用户的建议:

  1. 试试重新登录
  2. 换个浏览器
  3. 清一下浏览器缓存
  4. 如果还不行,告诉我们具体的报错信息

可能的原因:

  • 用户 session 过期
  • 前端 bug
  • 后端服务异常
  • 浏览器兼容性问题

内部 debug 路径:

  1. 让用户提供报错截图或浏览器控制台日志
  2. 看用户 session 状态
  3. 看后端服务是否正常

A.6 AI 助手参考规则

以下规则供 AskOLA 在加载本文档作为 skill 时使用:

  1. 优先回答用户友好的部分:即使用户问的是技术细节,也用"给用户的建议"开头,只在用户明确表示是技术人员/管理员时,才展开"内部 debug 路径"
  2. 不暴露代码路径:除非用户是已验证的内部技术人员,否则不要提及 backend/services/xxx.py 这种内部信息
  3. 不暴露具体技术栈:不要主动告诉用户我们用了 Gemini / OpenAI / 哪个数据库,除非用户明确询问
  4. 遇到不在本文档中的问题:诚实地告诉用户"这个问题我暂时无法直接解决,我会帮您转交给技术支持"
  5. 遇到涉及金额、合同、法律的问题:不要尝试自己回答,直接引导到客服或销售经理
